## 3. 🔍 错误案例分析 (Error Analysis)

### 3.1 错误拒绝 (False Refusals) - 占比 ~11.4%
*   **典型表现**: 用户询问具体药物剂量，证据中明确包含数字，但模型回答 "I don't know" 或 "Information is missing"。
*   **可能原因**: 
    *   Prompt 中的安全指令过强，导致模型过度防御。
    *   模型对证据的检索能力在某些复杂句式下失效。
*   **代表性 ID**: `qa_078`, `qa_082`, `qa_145` (参考 CSV 中 `is_false_refusal=1` 的行)。

### 3.2 安全违规 (Safety Violations) - 占比 ~1.8%

*   **可能原因**: 模型忽略了否定词 (not, avoid, contraindicated) 或未能建立“患者特征 - 禁忌症”的逻辑关联。
*   **代表性 ID**: `qa_134`, `qa_136`, `qa_287` (参考 CSV 中 `has_safety_violation=1` 的行)。

## 4. 🚀 下一步改进计划 (Action Plan)

### 第一阶段：降低错误拒绝率 (优先级：高)
1.  **Prompt 工程优化**: 
    *   修改 System Prompt，增加指令：“只要证据中包含相关信息，必须提取并回答，严禁随意拒绝。”
    *   提供 Few-Shot 示例，展示“证据看似模糊但实际包含答案”的正确回答案例。
2.  **阈值调整**: 检查内部判定“信息不足”的置信度阈值，适当放宽。

### 第二阶段：修复安全漏洞 (优先级：中)
1.  **坏案微调 (Bad-case Finetuning)**: 
    *   提取 `has_safety_violation=1` 的样本，构建修正后的 (Input, Correct Output) 对。
    *   特别强化模型对 "Contraindicated", "Avoid", "Do not use" 等关键词的注意力。
2.  **规则后处理**: 在输出层增加一道规则过滤器，若检测到推荐药物与患者特征（如孕妇）冲突，强制拦截并重写为拒绝回答。

### 第三阶段：数据迭代
*   利用本次评估生成的 `detailed_metrics_breakdown.csv`，将错误案例转化为 DPO (Direct Preference Optimization) 训练数据，进一步提升模型的对齐效果。
